Class Details

[line 121]
Basically implementation class of Xhwlay_Config_Interface.

It use global PHP-Array variable as configuration resource.

Example code is shown below. "(*)" mark indicates item can be omitted.

  1.  $configVars = array(
  2.      "story" => array(
  3.          "name" => "story name",
  4.          "bookmark" => "off", // (*):if omitted, default "on".
  5.          ),
  6.      "page" => array(
  7.          "page1.aci1" => array(
  8.              ...,
  9.              "bookmark" => "on", // (*):if omitted, story scope is assumed.
  10.              "next" => array(
  11.                  "page2" => "barrier1",
  12.                  "page3" => null,
  13.                  ),
  14.              "event" => array(
  15.                  "event1" => "guard1",
  16.                  "event2" => null,
  17.                  ),
  18.              ),
  19.          "page1.aci2" => array(...),
  20.          "page2.aci3" => array(...),
  21.          "page2.*" => array(...),
  22.          "page2" => array(...),
  23.          "*.*" => array(...),
  24.      "event" => array(
  25.          "event1" => array(
  26.              ...,
  27.              "transit" => array(
  28.                  "success" => "page2",
  29.                  "error" => "page4",
  30.                  ),
  31.              ),
  32.          ),
  33.      "barrier" => array(
  34.          "barrier1" => array(...),
  35.          "barrier2" => array(...),
  36.          ),
  37.      "guard" => array(
  38.          "guard1" => array(...),
  39.          "guard2" => array(...),
  40.          ),
  41.      );
  42.  $config =& new Xhwlay_Config_PHPArray($configVars);

Asterisk "*" means wildcad character.

  1.  "page1.user1" => array(...), // (1)
  2.  "*.user1" => array(...), // (2)
  3.  "*" => array(...), // (3)
If page is "page1" and aci is "user1", then (1). If page is not "page1" and aci is "user1", then (2). If page is not "page1" and aci is not "user1", then (3).

".aci" can be omitted. But be careful about priority.

  1.  "page1.user1" => array(...), // (1)
  2.  "page1.*" => array(...), // (2)
  3.  "page1" => array(...), // (3)
In this code, if page is "page1" and aci is "user1", then (1) will be page parameter, sure. If aci is default, then (2) will be, not (3). If (2) item is not defined and aci is default, (3) will be.

  1.  "*.*" => array(...), // (1)
  2.  "*" => array(...), // (2)
Priority is higher in (1) rather than (2).

More detailed and complicated pattern, see UnitTest source code.
