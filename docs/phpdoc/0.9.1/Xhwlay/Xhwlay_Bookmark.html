method destroy [line 277]

	

    

    

		void destroy(
)
    

    


	
		DESTROY this Bookmark.

If you call this, Bookmark data is destroyed only "on memory".  So, actual data still remains in Bookmark Container data storing  system, unless anyone calls "Bookmark Container"::save() method.
Even though you can restore Bookmarks by "Bookmark Container"::load()  method, we STRONGLY recommend once you call destroy(), you sholudn't  shouldn't touch destroyed Bookmark instance.
(destory() doesn't call its own clear() method, and, doesn't set   null to '$this'.)
