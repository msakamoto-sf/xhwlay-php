Xhwlay Hook System

This class supports "First-In-First-Called"(FIFC) hook callback invoking  feature.
Simple Usage is like below:  
  1.  $h =& Xhwlay_Hook::getInstance('sample');
    2. 

  2.  $h->pushCallback('hook1');
    3. 

  3.  $h->pushCallback('hook2');
    4. 

  4.  ...
    5. 

  5.  $h->invoke();
    6. 

  6.  $results = $h->allResult(true);
    7. 

  7.  var_dump($results[0]); // result of 'hook1'
    8. 

  8.  var_dump($results[1]); // result of 'hook2'
    9. 

Interface of hook-callback is like below:  
  1.  function func($hook_name, $arg1, $arg2, ...) { ... }
    2. 

  $hook_name is argument of getInstance(). If upper example,  'hook1' and 'hook2' retrieves 'sample' string value when invoking  'sample' hooks.  $arg1, $arg2, ... can specified by setArguments().  
  1.  $h =& Xhwlay_Hook::getInstance('sample');
    2. 

  2.  $h->setArgument(array($arg1, $arg2, ... ));
    3. 

Can I ENABLE/DISABLE actual invoking when calling invoke() ?  Yes. Set 'available' attribute to FALSE.  
  1.  $h =& Xhwlay_Hook::getInstance('sample');
    2. 

  2.  $h->setAttrubute('available', false);
    3. 

  3.  $h->pushCallback(...); // many times.
    4. 

  4.  ...
    5. 

  5.  $h->invoke();
    6. 

  6.  // Don't invoke any hook-callbacks actually.
    7. 

In one hook invoking, can I STOP following hooks and return
 from invoke()?  Yes. Call escape() in your hook code.  
  1.  function your_hook($hook_name) {
    2. 

  2.      $h =& Xhwlay_Hook::getInstance($hook_name);
    3. 

  3.      $h->escape();
    4. 

  4.  }
    5. 

  5.  $h =& Xhwlay_Hook('sample');
    6. 

  6.  $h->pushCallback('hook_before');
    7. 

  7.  $h->pushCallback('your_hook');
    8. 

  8.  $h->pushCallback('hook_after');
    9. 

  9.  $h->invoke();
    10. 

  Then, 'hook_before' and 'your_hook' is invoked, 'hook_after' not invoked.
How can I retrieve return values of hook callbacks ?  Use allResult() or some short cut methods like firstResult()/lastResult().  
  1.  function hook1($hook_name) { return 'value_1'; }
    2. 

  2.  function hook2($hook_name) {} // return null
    3. 

  3.  function hook3($hook_name) { return 'value_3'; }
    4. 

  4.  
    5. 

  5.  $h =& Xhwlay_Hook('sample');
    6. 

  6.  $h->pushCallback('hook1');
    7. 

  7.  $h->pushCallback('hook2');
    8. 

  8.  $h->pushCallback('hook3');
    9. 

  9.  $h->invoke();
    10. 

  10.  
    11. 

  11.  $results = $h->allResults(true);
    12. 

  12.  echo count($results); // 2
    13. 

  13.  echo count($results[0]); // 'value_1'
    14. 

  14.  echo count($results[1]); // 'value_2'
    15. 

  Notice that null return value is ignored in internal result stacks.  So, in this example, return value of 'hook2' (null) is not available  in allResults().  But you can change this behaviour and retrieve null value.  See next description.
How can I retrieve NULL value from hook callbacks?  Set 'ignore_null_result' attribute to TRUE.  
  1.  function hook1($hook_name) { return 'value_1'; }
    2. 

  2.  function hook2($hook_name) {} // return null
    3. 

  3.  function hook3($hook_name) { return 'value_3'; }
    4. 

  4.  
    5. 

  5.  $h =& Xhwlay_Hook('sample');
    6. 

  6.  $h1->setAttribute('ignore_null_result', true);
    7. 

  7.  $h->pushCallback('hook1');
    8. 

  8.  $h->pushCallback('hook2');
    9. 

  9.  $h->pushCallback('hook3');
    10. 

  10.  $h->invoke();
    11. 

  11.  
    12. 

  12.  $results = $h->allResults(true);
    13. 

  13.  echo count($results); // 3
    14. 

  14.  echo count($results[0]); // 'value_1'
    15. 

  15.  echo count($results[1]); // null
    16. 

  16.  echo count($results[2]); // 'value_2'
    17. 

OTHER NOTICE
  • Pushed callbacks remains after invoke().
  • allResult() don't clear result stack. If you want clear,
   call allResult(true).
  • Don't use reference in setArgument(). It leads
   'Call-time pass-by-reference' errors.
